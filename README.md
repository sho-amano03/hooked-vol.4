# HOOKED - Event Official Site
製作中のダンスイベント公式サイト。

## 概要

ストリートダンスを中心としたショーケースイベント「HOOKED」の公式サイトです。
ダンサーが作品（ショーケース）を製作し、当日ステージで発表します。
本イベントはこれまでに3回開催しており、現在は第4回（vol.4）の開催に向けて準備を進めています。
現在は主催側からダンサーへ出演オファーを行い、出演者情報やイベント詳細を随時更新しています。

## 目的

これまでイベント情報はInstagramを中心に発信していましたが、投稿が流れてしまい「必要な情報にたどり着けない等」課題がありました。
そこで、情報を整理して確認しやすい"公式サイト"を用意し、来場者・出演者が迷わず情報を得られる状態を目指しています。
また、イベント運営の効率化も目的の１つです。これまではチケット対応をDMで個別にやり取りしていましたが、今後はサイト上でチケット応募～管理まで完結できる導線を作る予定です。

## 技術工夫　/　学び

### 1. メニュー開閉を「class」で状態管理
JavaScriptで直接styleを書き換えるのではなく、`classList`でクラスを付け外しして表示状態を切り替えています。
CSSとJavaScriptの役割を分けることで、見た目の調整はCSS側だけで完結できるようにしました。

### 2. 「反映されない」問題の原因切り分け
実装中に「コードを書いたのに反映されない」問題が頻発したので、原因をパターン化して切り分けました。

- **誤字（class名 / id名　/ ファイル名）の確認**
  例：`querySelector`の対象が取れていない、CSSセレクタが一致していない　など
- **CSSの上書き（優先順位・同名セレクタ）の確認**
  例：別のCSSルールやメディアクエリが意図せず上書きしていた
- **transition等の指定が重複していないか確認**
  例：複数の要素/クラスに`transition`が設定され、想定外の挙動になる

#### 検証方法
Chrome DevTools を使用し、以下の流れで原因を特定しました。

1. Elementsタブで対象要素を選択し、Style / Computed を確認
2. CSSが打ち消し線になっていないかを確認（上書きの有無）
3. `console.log()`を用いてJavaScriptで要素を正しく取得できているか確認
4. 一時的に　`transition: none;`　を適用し、アニメーション由来の問題か切り分け

闇雲に書き直すのではなく、「表示されない理由を分解する」意識を持つことで、再現性のある修正ができるようになりました。
  
### 3. スマホ対応を前提にレイアウトを調整
モバイルでの閲覧を想定し、画面幅に応じてメニュー表示やレイアウトが崩れないよう調整しました。
（例：ハンバーガーメニュー化、余白、文字サイズの調整　など）
