# HOOKED - Event Official Site
本サイトは、実際のダンスイベント運用を想定し、情報整理とユーザー導線の最適化を目的として制作しました。

## このサイトで意識したこと
- イベント情報を整理し、来場者が必要な情報に迷わずたどり着ける構成にすること
- スマートフォンでも見やすいレイアウトにすること
- JavaScriptを使って、ハンバーガーメニューなどの基本的なUI操作を実装すること

## 🔗 Demo
https://sho-amano03.github.io/hooked-vol.4/

## 概要

ストリートダンスを中心としたショーケースイベント「HOOKED」の公式サイトです。
ダンサーが制作した作品をステージで発表するイベントで、これまで3回開催されています。
現在は第4回(vol.4)の開催に向けて準備中です。

## 制作背景 / 目的

これまでイベント情報はInstagramで発信していましたが、情報が流れてしまい必要な情報にたどり着きにくい課題がありました。
そのため、情報を整理し来場者・出演者が迷わず確認できる公式サイトを制作しました。

また、チケット対応をDMで行っていたため、今後はサイト上で応募〜管理まで完結できる導線の構築も目的としています。

## ページ構成 / 目的

### index.html（トップページ）
- イベント概要
- 開催日時
- 会場情報
- 料金案内
- 過去開催の写真（past events / vol.1~3）
- 各ページ（Cast/Ticket/Timetable）への導線

来場者が必要情報を把握しやすい構成を意識しました。

### cast.html（CASTページ）
- 出演者一覧
 - DJ＆MC
 - Pickup Dancer
 - Dancer
 - Guest

 来場者が目当ての出演者を探しやすいようカテゴリ分けを行いました。
 また、将来的に来場者の目的(誰目当てか)を把握できる設計を意識しています。

 ### ticket.html（Ticketページ）
 - チケット情報の掲載予定
 - 応募導線（現在未実装）

 現在はInstagramのDMで応募管理を行っていますが、
 今後はサイト上で応募〜管理まで完結できるよう機能拡張を予定しています。

 ### timetable.html（Timetableページ）
 - 当日のタイムテーブル
 - 進行順・出演順の可視化

来場者が目当ての出演時間を把握しやすくすること、
出演者側も当日の進行を確認できるよう、シンプルな構成を意識しました。

※モバイル閲覧を前提に、見やすさを意識したレイアウト設計を行っています。

 ## 使用技術
 - HTML5

 - Git / GitHub

JavaScriptは基礎理解を深めるため、フレームワークを使用せずに実装しました。
ハンバーガーメニューの開閉など、基本的な動作を中心に実装しています。

## 技術工夫　/　学び

■メニュー開閉の実装  
- 課題：メニューの表示/非表示の状態管理  
- 対応：classListを用いてクラスの付け外しで制御し、見た目はCSSで管理  

■スタイルが反映されない問題  
- 課題：CSSが適用されない  
- 対応：DevToolsでセレクタの不一致やCSSの上書きを確認し、原因を特定  

■アニメーションの不具合  
- 課題：transitionが意図通り動作しない  
- 対応：複数箇所に設定されていたtransitionを整理し、適用箇所を統一  

■レスポンシブ対応  
- 課題：スマホ表示でレイアウトが崩れる  
- 対応：メディアクエリを用いて、余白や文字サイズを調整
