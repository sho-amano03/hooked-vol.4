# HOOKED - Event Official Site
製作中のダンスイベント公式サイト。

## 🔗 Demo
https://sho-amano03.github.io/hooked-vol.4/

## 概要

ストリートダンスを中心としたショーケースイベント「HOOKED」の公式サイトです。
ダンサーが作品（ショーケース）を製作し、当日ステージで発表します。
本イベントはこれまでに3回開催しており、現在は第4回（vol.4）の開催に向けて準備を進めています。
現在は主催側からダンサーへ出演オファーを行い、出演者情報やイベント詳細を随時更新しています。

## 本サイトの目的

これまでイベント情報はInstagramを中心に発信していましたが、投稿が流れてしまい「必要な情報にたどり着けない等」課題がありました。
そこで、情報を整理して確認しやすい"公式サイト"を用意し、来場者・出演者が迷わず情報を確認しやすい状態を目指しています。
また、イベント運営の効率化も目的の１つです。
これまではチケット対応をDMで個別にやり取りしていましたが、今後はサイト上でチケット応募～管理まで完結できる導線を作る予定です。

## ページ構成、目的

### index.html（トップページ）
- イベント概要
- 開催日時
- 会場情報

- 各ページ（Cast/Ticket/Timetable）への導線

来場者が最短で必要情報を把握できる構成を意識しました。

- 出演者一覧
 - DJ＆MC

 - Guest

 ### ticket.html（Ticketページ）

また出演者側も当日の進行を確認できるよう、シンプルな構成を意識しました。

スマホでも見やすくなるよう、シンプルで分かりやすいレイアウトを意識しました。

## 技術工夫　/　学び

### 1. メニュー開閉を「class」で状態管理
JavaScriptでボタンをクリックした時に、メニューの表示・非表示を切り替える処理を実装しました。
classListを使ってクラスを付け外しすることで、表示状態を切り替えています。
見た目の変化はCSSで制御しています。

### 2. 「反映されない」問題の対応
実装中に「コードを書いたのに反映されない」問題が多く発生したため、原因を１つずつ確認しました。

- **誤字（class名 / id名　/ ファイル名）の確認**
  例：`querySelector`の対象が取れていない、CSSセレクタが一致していない　など
- **CSSの上書き（優先順位・同名セレクタ）の確認**
  例：別のCSSルールやメディアクエリが意図せず上書きしていた
- **transition等の指定が重複していないか確認**
  例：複数の要素/クラスに`transition`が設定され、想定外の挙動になる

#### 検証方法
最初は原因がわからず時間がかかりましたが、
Chrome DevTools を使いながら、１つずつ原因を確認しました。

・ 要素を選択して、適用されているCSSを確認
・ 意図したスタイルが反映されているかチェック
・ console.log()で要素を正しく取得できているか確認

なんとなく直すのではなく、原因を１つずつ確認することで解決できるようになりました。


  
### 3. スマホ対応を前提にレイアウトを調整
モバイルでの閲覧を想定し、画面幅に応じてメニュー表示やレイアウトが崩れないよう調整しました。
（例：ハンバーガーメニュー化、余白、文字サイズの調整　など）
